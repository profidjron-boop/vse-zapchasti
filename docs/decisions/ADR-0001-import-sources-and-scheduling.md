# ADR-0001: Import Sources & Scheduling (1C-ready)

**Status:** Accepted  
**Date:** 2026-03-04  
**Project:** Все запчасти  
**Context:** Каталог/цены/остатки должны быть актуальными. Сейчас 1С не подключена, но требуется заложить интеграцию “по максимуму” без переписывания ядра.

## Decision
1) Вводим единый **import pipeline** (нормализация данных → применение в каталог → отчет ошибок).
2) Поддерживаем несколько источников (`import_source.type`):
   - `file_upload` (ручная загрузка CSV/Excel/CommerceML)
   - `commerce_ml` (обмен CommerceML из 1С)
   - `api_1c` (сетевой API/обмен, если будет)
3) Витрина (public каталог/поиск/карточка) читает только из **normalized catalog**, не зависит от формата 1С.
4) Обновление актуальности обеспечивается планировщиком:
   - “полный импорт” (по расписанию)
   - “дельта цен/остатков” (по расписанию)
   Конкретные интервалы задаются в настройках, дефолт будет определён позже.

## Consequences
- Можно стартовать с ручной загрузки, а затем подключить 1С “по сети” без изменения публичной части и процессов обработки заявок.
- Появляется трассировка импортов и прозрачные отчеты ошибок.
- Требуется модуль админки “Импорты” + хранение результатов прогонов.

## Non-goals (v1)
- Полный двусторонний обмен с 1С.
- Глубокая синхронизация заказов/документов.
