# UI Direction — Все запчасти

**UI profile:** industrial-commerce
**Theme:** premium light with industrial accents
**Visual mode (public):** enabled
**Runtime policy:** NO CDN, все ассеты self-hosted.

## Core direction
- Публичный сайт оформляется как современный магазин запчастей и сервисный центр, а не как SaaS-панель.
- Базовая палитра: светлый фон, графитовые нейтрали, фирменный синий как основной бренд-цвет, теплый оранжевый только для действий и статусов.
- Интерфейс должен оставаться плотным и практичным: меньше “пустоты ради красоты”, больше полезной структуры, ясных секций и понятных точек входа.

## Brand tone
- Уверенно, коротко, по делу.
- Акцент на скорости подбора, наличии, понятной записи на сервис и работе с коммерческим транспортом.
- CTA короткие и конкретные: “Подобрать”, “Записаться”, “Оставить VIN”, “В корзину”.

## Public shell
- Единая шапка на всех публичных страницах.
- Desktop:
  - логотип слева,
  - основной поиск и быстрые действия по центру/справа,
  - главная навигация на отдельной ровной линии,
  - sticky header без визуальных скачков между страницами.
- Mobile:
  - компактная шапка,
  - упрощенная навигация,
  - CTA и ключевые действия остаются доступными без потери сценариев.
- Footer единый для всех публичных страниц: контакты, разделы каталога, сервис, правовые страницы.

## Homepage must-have
1) Hero:
   - H1 + подзаголовок
   - 2 CTA:
     - “Подобрать запчасти”
     - “Записаться на сервис”
   - быстрый поиск или точка входа в каталог
2) Быстрые сценарии:
   - поиск по артикулу/OEM,
   - VIN-заявка,
   - сервисная запись.
3) Крупные категории запчастей:
   - плитки с иконкой/изображением,
   - явный переход в каталог.
4) Блок “Сервис”:
   - направления работ,
   - преимущества,
   - быстрый CTA.
5) Блок доверия:
   - преимущества,
   - цифры,
   - гарантии/поддержка.
6) Контакты:
   - адрес,
   - режим работы,
   - телефоны,
   - self-hosted map preview / схема.

## Catalog direction
- Каталог должен выглядеть как магазин:
  - слева фильтры и категории,
  - справа сетка карточек товаров,
  - сверху поиск, сортировка и контекст категории.
- Карточка товара в листинге:
  - изображение,
  - название,
  - артикул/OEM,
  - цена или “Цена по запросу”,
  - наличие,
  - действия: карточка / корзина / заявка.
- Не использовать визуально слабые “табличные” списки вместо карточек.

## Product page direction
- Крупный товарный блок сверху.
- Слева изображение/галерея, справа название, бренд, SKU/OEM, цена, наличие, CTA.
- Ниже: характеристики, описание, совместимость, связанные действия.

## Service direction
- Сервисные страницы должны быть визуально согласованы с каталогом.
- Акцент на доверии, понятных видах работ и быстрой записи.
- Не использовать неработающие переключатели и декоративные элементы без сценария.

## Components and states
- Все интерактивы имеют единые состояния:
  - default
  - hover
  - active
  - focus-visible
  - disabled
  - loading
- Все страницы и ключевые блоки имеют:
  - loading
  - empty
  - error
  - success

## Interaction & motion
- Hover/press для всех кнопок, ссылок, карточек и табов.
- Плавные переходы только через opacity/transform.
- Никакой декоративной анимации, мешающей восприятию каталога.

## Typography & spacing
- Контейнер: 1200–1280px на desktop.
- Крупный hero, но без перегруза.
- Сетка секций плотнее и взрослее, чем сейчас.
- Единый ритм отступов и размеров карточек на всех публичных страницах.
