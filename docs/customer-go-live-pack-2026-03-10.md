# Пакет для заказчика: запуск проекта «Все запчасти»

Дата: 2026-03-10  
Версия: v1.1  
Статус: pre-prod / release-candidate

Назначение документа:
- дать заказчику понятную картину готовности,
- описать как пользоваться системой,
- показать как включаются функции без правки кода,
- зафиксировать, что ещё нужно от заказчика для боевого запуска.

---

## 1) Текущее состояние проекта

Проект технически готов к запуску в production при наличии production-доступов и заполненных метаданных внедрения.

Подтверждено:
- публичный сайт работает,
- админка работает,
- ключевые сценарии (каталог, корзина, заявки, сервис, импорты) работают,
- release-check/verify/runtime проверки проходят,
- backup/restore path подтвержден,
- в админке есть живые уведомления по новым обращениям (включая VIN-заявки).

Ограничения по умолчанию:
- online-платежи не включены,
- full online-sync адаптер 1С/ERP требует отдельного этапа,
- боевые провайдеры уведомлений требуют ключей/доступов заказчика.

## 2) Полное описание функций сайта

## 2.1 Публичная часть
Доступные разделы:
- Главная `/`
- Каталог `/parts`
- VIN-подбор `/parts/vin`
- Автосервис `/service`
- Корзина `/cart`
- Избранное `/favorites`
- Мои заказы `/account/orders`
- О компании `/about`
- Контакты `/contacts`

Что умеет сайт:
- поиск товаров по SKU/OEM/названию,
- подбор по параметрам авто,
- переход категория → подкатегория → товар,
- карточка товара с основными параметрами, ценой и совместимостью,
- гостевая корзина без регистрации,
- оформление заказа (самовывоз/курьер, при получении/по счёту),
- быстрый заказ,
- избранное,
- история заказов по телефону,
- VIN-заявка менеджеру,
- заявка на сервис,
- связка «запчасть + установка» (как заявка, подтверждение менеджером).

## 2.2 Админка
Доступные разделы:
- Дашборд `/admin`
- Редактор сайта `/admin/content`
- Отчёты `/admin/reports`
- Импорты `/admin/imports`
- Товары `/admin/products`
- Категории `/admin/categories`
- Пользователи `/admin/users`
- Интеграции `/admin/integrations`
- Заявки (запчасти) `/admin/leads`
- VIN-заявки `/admin/vin-requests`
- Заказы `/admin/orders`
- Заявки (сервис) `/admin/service-requests`
- Справочник услуг `/admin/service-catalog`

Что умеет админка:
- управлять товарами/категориями,
- загружать и запускать импорты,
- обрабатывать заявки и заказы,
- менять статусы,
- смотреть отчёты,
- редактировать контент страниц,
- управлять ролями пользователей,
- включать интеграционные функции feature-флагами,
- получать живые уведомления о новых обращениях:
  - общий счётчик в шапке,
  - колокольчик,
  - бейджи в левом меню,
  - опциональный звуковой сигнал.

## 3) Как настраивать (без кода)

Основная страница настройки: `/admin/integrations`.

## 3.1 1С/ERP
Через UI можно:
- разрешить импорт из источника,
- включить online-sync,
- включить delta и retry,
- задать URL источника, allowlist хостов,
- задать интервал и параметры retry,
- запускать sync вручную и смотреть статус.

## 3.2 Уведомления
Через UI можно:
- включить уведомления глобально,
- включить/выключить каналы Email/SMS/мессенджер,
- включить очередь и retry,
- проверять health,
- отправлять тестовое уведомление.

## 3.3 Предоплата сервиса
Через UI можно:
- включить отображение предоплаты,
- включить payment-flow,
- включить блокировку неоплаченных сценариев,
- проверять health платежного контура.

---

## 4) Что нужно настроить через DevOps/.env

Через UI не хранятся секреты. Для production обязательны env-настройки:
- домены сайта/API,
- JWT/DB/Redis секреты,
- SMTP и webhook ключи уведомлений,
- секреты/аутентификация источника 1С/ERP,
- cookie security параметры.

Подробно: `docs/deploy.md`.

---

## 5) Что обязательно нужно от заказчика

## 5.1 Блокеры запуска
- утверждённые ответственные и эскалация,
- production домены и DNS-доступ,
- production серверные доступы,
- место и регламент хранения секретов,
- утверждённые юридические тексты (политика, оферта, согласие).

## 5.2 Для полного боевого режима
- выбранные провайдеры Email/SMS/мессенджер + ключи,
- выбранный протокол и доступы к 1С/ERP,
- если нужна предоплата: выбранный платёжный провайдер + согласованный 54‑ФЗ контур + возвраты.

## 5.3 Контент
- финальные тексты и медиа,
- проверенные контакты и реквизиты,
- подтверждённый прайс/каталог/услуги.

Отдельный документ-чеклист: `docs/customer-go-live-required-inputs-2026-03-10.md`.

## 6) Роли и процесс работы после запуска

Рекомендуемый поток:
1. Менеджер видит новые обращения по счётчикам/колокольчику.
2. Открывает карточку заявки/заказа.
3. Меняет статус (`new -> in_progress -> closed` для заявок, расширенная схема для заказов).
4. При необходимости связывается с клиентом.
5. Руководитель контролирует отчёты и SLA реакции.
